Week 2 Project: Semantic HTML and CSS Website
Description
This project is a simple, responsive webpage built using semantic HTML and CSS. It demonstrates the use of proper HTML structure with elements like <header>, <nav>, <main>, <section>, and <footer>, along with CSS for styling, layout, and responsiveness. The site includes three main sections: Home, About, and Contact, with a navigation menu and a contact form.

Features
Semantic HTML Structure: Uses semantic tags for better accessibility and SEO.
Responsive Design: Includes media queries to adapt the layout for smaller screens (e.g., mobile devices).
Styled Navigation: Horizontal menu on desktop, vertical on mobile, with hover effects.
Form Handling: Basic contact form with labels and inputs.
Visual Enhancements: Background colors, shadows, border-radius, and typography for improved usability.
Technologies Used
HTML5

Reflections
HTML Challenges
Organizing the semantic structure required careful consideration of element usage. For example, choosing <section> over <article> for thematic groupings like Home, About, and Contact ensured logical content organization. Nesting elements, such as the form in the Contact section with proper label-input pairings, took practice to maintain accessibility. Structuring the navigation as an unordered list with links inside list items provided semantic meaning but required planning for CSS interactions.

CSS Improvements
CSS transformed the basic HTML into a visually appealing design. Color schemes (e.g., green header and dark navigation) established hierarchy, while spacing and padding improved readability. Shadow effects and border-radius added modern depth, and hover effects on navigation links enhanced usability by providing clear feedback. Typography adjustments, like increased line-height, made text more readable across devices.

Git/GitHub Experience
Learning Git commands like git add, git commit, and git push was initially confusing due to the staged workflow. Practicing with frequent commits (e.g., after adding HTML structure or CSS) helped solidify understanding. No merge conflicts occurred in this solo project, but clear commit messages (e.g., "Add responsive media queries") made version control valuable for tracking changes and potential rollbacks. Overall, it proved helpful for managing project progress.
